## Phase 7: Production Tools & Static Binary

### Objective
Build a completely standalone `wafter-musl` binary that runs on any Linux system without external dependencies. Also implement advanced features for packet analysis and filtering.

### Architecture Overview

```
┌─────────────────────────────────────────────┐
│ Developer: make linux                       │
│ (or: docker build)                          │
└─────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────┐
│ Docker Build (multi-stage)                  │
│ ├─ Base: jerboa21/jerboa                   │
│ ├─ Compile all .ss modules                 │
│ ├─ Generate C entry point                  │
│ ├─ Link statically with musl               │
│ └─ Strip and verify                        │
└─────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────┐
│ wafter-musl (Fully Static Binary)           │
│ ├─ No glibc dependency                      │
│ ├─ No Chez Scheme runtime needed            │
│ ├─ No Jerboa libraries needed               │
│ └─ ~40-60MB executable                      │
└─────────────────────────────────────────────┘
```

## Notes

### musl libc Requirement
Static binary requires musl-built Chez Scheme. The `jerboa21/jerboa` Docker image includes this, making Docker the canonical build path.

For local development without Docker:
```bash
# Install musl toolchain
sudo apt install musl-tools

# Build Chez with musl (one-time setup, ~30 min)
cd ~/chez-scheme
./configure --threads --static CC=musl-gcc
make && make install DESTDIR=~/chez-musl

# Then: make linux-local
```

### Docker Image Choice
Using `jerboa21/jerboa` (maintained by community):
- Pre-built musl Chez
- Pre-installed Jerboa
- Build tools (gcc, binutils, etc.)
- Regular maintenance expected

**If image unavailable**: Can build custom base using similar pattern.
